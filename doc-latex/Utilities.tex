\chapter{Utilities} \label{chap:Utils}

All utilities have command line options affecting their behaviour. The following
options are common for many utilities, so they are described here rather than at
the individual utilities where only list of the possible options is
provided.\\[1em]
\noindent
\begin{tabular}{p{0.14\textwidth}p{0.804\textwidth}}
  \toprule
  \multicolumn{2}{l}{General options}\\
  \midrule
  \tt{--verbose} & print information about all bead and molecule types to the
                   screen\\
  \tt{--silent}  & run silently, i.e., without any output at all (overrides
                   \tt{--verbose} option)\\
  \tt{--help}    & print short description of the utility and usage\\
  \tt{--version} & print version of the utilities and exit\\
  \midrule
  \multicolumn{2}{l}{Options for structure and coordinate input files}\\
  \midrule
  \tt{-i <name>}  & use specified structure file, see
    \cref{sec:StructFile} for possible formats\\
  \tt{-st <n>}    & start calculations at \tt{n}-th
                    timestep\\
  \tt{-e <n>}     & end calculations at \tt{n}-th
                    timestep\\
  \tt{-sk <n>}    & skip every \tt{n} timesteps (i.e., use every \tt{(n+1)}-th
                    timestep)\\
  \bottomrule
\end{tabular}\\[1em]

% For \vtf coordinate file with indexed timesteps, all timesteps are expected to
% have the same number of beads (though not necessarily the same beads). If some
% timesteps contain different number of beads, \tt{--variable} option must be
% used. The option is implemented because without knowing in advance the number of
% beads in a timestep, reading that timestep is slower (significantly so for large
% systems).

% Plain \xyz format provides only bead names and coordinates, but the comment line
% at the beginning of every timestep can store extra information. Here, only box
% dimensions are read, but because there are extended \xyz formats containing more
% information, the \tt{-pbc} option is used to specify where the box dimensions
% are (as well as to tell the utility there are box dimensions at all). For
% example, \tt{-pbc 3} must be specified for the following comment line

% \tt{other text 10 10 10 90 60 30}.

% \noindent
% This would define a paralleloid box with $a=b=c=10$ and angles
% $\alpha=90^\circ$, $\beta=60^\circ$, and $\gamma=30^\circ$. If less than 6
% numbers were provided, only the first three would be used to define a cuboid
% box.

% Which timesteps are used by the utility can be controlled via the \tt{-st},
% \tt{-e}, and \tt{-sk} options. For example, specifying \tt{-st 2 -e 10 -sk 2}
% would use timesteps 2, 5, and 8.

The \tt{-st}, \tt{-e}, and \tt{-sk} options can be combined, so that, for
example, specifying \tt{-st 2 -e 10 -sk 2} would use timesteps 2, 5, and 8.

Note that should two (or more) identical options be specified on the command
line, only the first one is used.
